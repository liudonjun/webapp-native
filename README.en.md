## üîß Requirements

### Required Environment

- **Node.js** >= 16.0.0
- **npm/yarn/pnpm** (pnpm recommended)
- **Android SDK** (installed via Android Studio)
- **Java JDK** 11+ (JDK 17 recommended)

### Android SDK Configuration

1. Install [Android Studio](https://developer.android.com/studio)
2. Install Android SDK via Android Studio (API Level 34 recommended)
3. Set environment variables:
   - **Windows**: Set system environment variable `ANDROID_HOME` = `C:\Users\YourName\AppData\Local\Android\Sdk`
   - **macOS/Linux**: Add to `~/.bashrc` or `~/.zshrc`:
     ```bash
     export ANDROID_HOME=$HOME/Library/Android/sdk
     export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools
     ```

### Java JDK Configuration

1. Install JDK 11 or higher
2. Set environment variable `JAVA_HOME`:
   - **Windows**: `C:\Program Files\Java\jdk-17`
   - **macOS/Linux**: `/Library/Java/JavaVirtualMachines/jdk-17.jdk/Contents/Home`

## üöÄ Quick Start

### First Time Setup (Complete Process)

#### Step 1: Install Dependencies

```bash
npm install
# or
pnpm install
```

#### Step 2: Create Environment Configuration File

```bash
npm run create:env
```

**What it does**:
- Auto-detect Android SDK and Java SDK paths
- Generate `.env` file based on `.env.example`
- Configure basic environment variables

**Output**: Generates `.env` file

#### Step 3: Edit Environment Configuration (Optional)

Edit the `.env` file to configure app information:

```env
# Basic app information
APP_ID=com.example.app
APP_NAME=My App
APP_VERSION=1.0.0

# Android SDK path (auto-detected)
ANDROID_HOME=C:\Users\YourName\AppData\Local\Android\Sdk

# Build mode
BUILD_MODE=debug
```

#### Step 4: Initialize Project

```bash
npm run init
```

**What it does**:
- Detect frontend framework type (React/Vue/Angular/Vanilla)
- Initialize Capacitor configuration
- Create Android project structure
- Configure build environment

**Output**:
- `capacitor.config.json` - Capacitor configuration
- `android/` - Android project directory

#### Step 5: Develop H5 Project

Develop your frontend project in the `web/` directory:

```bash
npm run dev
```

**What it does**: Start development server (Vite) with hot reload

**Access**: Open `http://localhost:3000` in browser

#### Step 6: Build APK

```bash
npm run build:apk
```

**What it does**: Execute complete build process (see [Build Process](#build-process) below)

**Output**: `build/app-debug.apk` or `build/app-release.apk`

---

# 1. Start development server

## üìñ Command Reference

### Initialization Commands

#### `npm run init`

**Function**: Project initialization

**Execution**:
1. Detect frontend framework type (by checking `web/package.json`)
2. Create necessary directory structure
3. Create basic web project files (if they don't exist)
4. Update `capacitor.config.json` configuration
5. Initialize Capacitor Android project

**Use Cases**:
- First time using the project
- Re-initialize project structure

**Prerequisites**:
- Dependencies installed (`npm install`)
- `.env` file created (recommended)

---

### Environment Configuration Commands

#### `npm run create:env`

**Function**: Create environment variable configuration file

**Execution**:
1. Detect Android SDK path
2. Detect Java SDK path
3. Generate `.env` file based on `.env.example`
4. Auto-fill detected paths

**Output**: `.env` file

**Use Cases**:
- First time configuring project
- Changing development environment

---

#### `npm run create:env:example`

**Function**: Create environment variable template file

**Execution**: Generate `.env.example` template file

**Output**: `.env.example` file

**Use Cases**:
- Creating project template
- Documentation maintenance

---

### Development Commands

#### `npm run dev`

**Function**: Start H5 development server

**Execution**:
- Switch to `web/` directory
- Run `npm run dev` (Vite development server)
- Start hot reload development environment

**Access URL**: `http://localhost:3000`

**Use Cases**:
- Daily frontend development
- Debugging H5 features

---

### Build Commands

#### `npm run build:web`

**Function**: Build H5 project

**Execution**:
1. Detect build command (npm/yarn/pnpm)
2. Execute frontend build command (e.g., `npm run build`)
3. Validate build output (check `web/dist/` directory)
4. Check if `index.html` exists

**Output**: `web/dist/` directory

**Use Cases**:
- Build H5 project separately
- Verify frontend build success

**Prerequisites**:
- `web/` directory exists
- `web/package.json` has `build` script

---

#### `npm run sync`

**Function**: Sync H5 build artifacts to Capacitor

**Execution**:
1. Verify `web/dist/` directory exists
2. Update `capacitor.config.json` configuration
3. Update Android app name (`strings.xml`)
4. Execute `npx cap sync android`

**Output**:
- Updated `capacitor.config.json`
- Updated `android/app/src/main/res/values/strings.xml`
- Synced files to `android/app/src/main/assets/`

**Use Cases**:
- After H5 build, sync to Android project
- After updating app configuration, re-sync

**Prerequisites**:
- `npm run build:web` has been run
- `web/dist/` directory exists and has content

---

#### `npm run build:android`

**Function**: Compile Android APK

**Execution**:
1. Verify Android project exists
2. Detect Java SDK and Android SDK
3. Configure `local.properties` (SDK path)
4. Update Android configuration (`build.gradle`):
   - `applicationId`
   - `versionCode` and `versionName`
   - `minSdkVersion` and `targetSdkVersion`
   - Signing configuration (Release mode)
5. Update app name (`strings.xml`)
6. Execute Gradle build:
   - `gradlew clean`
   - `gradlew assembleDebug` or `gradlew assembleRelease`
7. Copy APK to `build/` directory

**Output**:
- `android/app/build/outputs/apk/debug/app-debug.apk`
- `android/app/build/outputs/apk/release/app-release.apk`
- `build/app-debug.apk` or `build/app-release.apk`

**Use Cases**:
- Compile APK separately
- Debug Android build issues

**Prerequisites**:
- `npm run sync` has been run
- Android project initialized

---

#### `npm run build:apk`

**Function**: Complete build process (one-click build)

**Execution Order**:
```bash
npm run build:web    # 1. Build H5 project
npm run sync         # 2. Sync to Capacitor
npm run build:android # 3. Compile APK
```

**What it does**: One-click completion of complete build process from H5 to APK

**Output**: `build/app-debug.apk` or `build/app-release.apk`

**Use Cases**:
- Daily APK building
- CI/CD automated builds

---

### Signing Commands

#### `npm run generate:keystore`

**Function**: Generate Android signing certificate

**Execution**:
1. Interactive prompts for certificate information:
   - Keystore file path
   - Keystore password
   - Key alias
   - Key password
   - Certificate validity period
   - Certificate information (name, organization, etc.)
2. Use `keytool` to generate keystore file
3. Automatically update signing configuration in `build.config.js` or `.env`

**Output**:
- Keystore file (e.g., `release.keystore`)
- Updated configuration files

**Use Cases**:
- First time generating signing certificate
- Creating new signing key

**Prerequisites**:
- Java JDK installed
- `keytool` command available

---

### Debug vs Release Build

#### Debug Build

```bash
# Method 1: Use .env configuration
# BUILD_MODE=debug
npm run build:apk

# Method 2: Default is debug mode
npm run build:apk
```

**Characteristics**:
- Unsigned
- Contains debug information
- Larger file size
- Suitable for development and testing

#### Release Build

```bash
# Method 1: Use .env configuration
# BUILD_MODE=release
# SIGNING_ENABLED=true
npm run build:apk

# Method 2: Use build.config.js
# buildMode: 'release'
# signing.enabled: true
npm run build:apk
```

**Characteristics**:
- Signed
- Code obfuscation (optional)
- Optimized file size
- Suitable for release

**Prerequisites**:
- Signing certificate generated (`npm run generate:keystore`)
- Signing information configured (`.env` or `build.config.js`)

## ‚ùì FAQ

### 1. Android SDK Not Found

**Error Message**:
```
SDK location not found. Define a valid SDK location...
```

**Solution**:
1. Ensure Android Studio is installed
2. Run `npm run create:env` to auto-detect SDK path
3. Or manually set `ANDROID_HOME` environment variable

---

### 2. Java Version Incompatible

**Error Message**:
```
Unsupported Java version
```

**Solution**:
1. Install JDK 11 or higher (JDK 17 recommended)
2. Set `JAVA_HOME` environment variable
3. Run `npm run create:env` to auto-detect

---

### 3. Gradle Build Failed

**Possible Causes**:
- Network issues (Gradle needs to download dependencies)
- Android SDK not correctly installed
- Configuration errors

**Solution**:
1. Check network connection
2. Check `sdk.dir` path in `android/local.properties`
3. View detailed error logs
4. Refer to [Gradle Mirror Configuration](docs/guide/gradle-mirror.md)

---

### 4. Capacitor Sync Failed

**Error Message**:
```
Web build output does not exist
```

**Solution**:
1. Ensure `npm run build:web` has been run
2. Check if `web/dist/` directory exists
3. Check if `web/dist/index.html` exists
4. Check `capacitor.config.json` configuration

---

### 5. App Name Not Updated

**Issue**: After modifying `APP_NAME` in `.env`, app name in APK is not updated

**Solution**:
1. Ensure `npm run sync` has been run (updates `strings.xml`)
2. Re-run `npm run build:apk`
3. Check `android/app/src/main/res/values/strings.xml`

---

### 6. Release Version Signing Failed

**Error Message**:
```
Keystore file not found
```

**Solution**:
1. Run `npm run generate:keystore` to generate certificate
2. Check if signing configuration in `.env` is correct
3. Ensure `KEYSTORE_PATH` path is correct

---

### 7. Tests Failed

**Issue**: Tests fail when running `npm test`

**Solution**:
1. Ensure all dependencies are installed (`npm install`)
2. Check if `.env` file affects tests (tests mock dotenv)
3. View test error messages
4. Refer to `tests/README.md`
